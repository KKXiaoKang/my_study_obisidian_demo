## 核心问题：SFT训练到什么程度，才值得做RL？

（前期的SFT对目标几乎是随机策略的模型可以有指导作用，避免模型无边界的试错）

- 分析现有SFT不足之处
    
    - SFT训的好，本质上是模型的输出分布 = 现实环境的输出分布是相等的 （以DeepSeek V2作为例子，数据分为了150条高质量SFT数据，其中120万条有益数据，30万条安全数据，学习率控制在5x10e-6的程级，在2个epoch下接近收敛）
        
    - 但是模型只学会了怎么做，但是不会做的更好。
        
        - 最关键的判断标准：或者说你用你的reward采样分布（对于LLM来说，当你用SFT后的模型给定同一个prompt采样多个回答时，如果reward既有高分又有低分，说明模型已经可以有效果，但是质量参差不齐，这时候上RL才有效果）
            
        - 或者换言之Robot在policy eval的时候动作做的好不好，有没有什么无效动作，或者不安全的动作，这时候上RL会更合适，尤其是某些精细化程度高的场景下，模型如果在实际表现上更容易产生负反馈导致失败，这时候上RL也会更合适
            
        - 其实正如（https://arxiv.org/pdf/2504.13837）的RLVR的结论
            
            - SFT专攻Pass@k，RL强化Pass@1
                
        - 整体下切RL提高场景成功率应该要有两个指标
            
            - 动作的reward分布相对比较均匀（有一定的成功率70%，RL需要agent能获得足够的正向反馈才能学习）
                
            - 在多套轨迹动作中要找到最优轨迹（提升动作效率）

- 发现kai0的路线只使用了dagger + 优势估计器的特性进行训练

        - 使用初始数据集训练策略模型
            
        - 模型在真实机器人上执行任务
            
        - 操作员观察模型行为，发现错误
            
        - 进入DAgger模式（按'd'键）
            
            - 暂停模型推理
                
            - 切换到人工控制模式
                
        - 人工演示纠正
            
            - 操作员通过主臂（master arm）演示正确行为
                
            - 系统记录：observation + 人工动作 + intervention=1

        - 恢复推理（按'r'键）
            
            - 退出DAgger模式
                
            - 继续使用模型控制（intervention=0）
                
        - 将新收集的DAgger数据合并到数据集
            
            - 回到步骤 _优势估计器_ ，形成改进循环

        - DAgger采集的数据可以直接过这个已经先前用大量高质量数据训好的优势估计器，自动给出推理标注（离散化为task_index | task_index=0: 低优势（bottom 70%）| task_index=1: 高优势（top 30%））
            
        - 然后重新合并到优势数据集
            
    - AWBC优势加权实现: 基于数据集优势加权学习，actor更新 更保守，更新的a必须是dataset里面的a。（传统的SAC因为带熵项，容易出现一些不在分布内的a导致exploitation error）

- 重点讲解&简要的概括一下每步的训练思路：
    
- pre-train model
    
    - 准备多任务离线数据集 | 训练value head Function network（就是Critic 网络轻量化VLA）
        
        - 多机器人的离线数据集，包含观察 (o_t)、动作 (a_t)、奖励 (r_t)、任务标签 (ℓ), 一般用稀疏奖励sparse reward
            
        - 训练value head Function： 通过MSE监督学习预训练时使用整个 episode 的累积奖励
            
    - 计算 Advantage动作优势值 并 二值化 | 训练 Advantage-Conditioned Policy（actor policy）
        
        - 计算Advantage
            
            - 上述训练好的 Critic 网络轻量化VLA 输出 _A_∈R（连续值）
                
            - 然后二值化 I ∈{0,1}（离散值）（关于二值化的阈值，论文 Section III-F 提到预训练时：选择阈值 ε_ℓ 使得**约 30%** 的演示数据有正优势， 基于随机采样的 10k 数据点计算 | ）
                
            - 用于策略训练：策略 conditioning 在二值化的 I 上
                
        - 训练 Advantage-Conditioned Policy（actor policy）
            
            - 使用 **flow-matching loss**（因为动作头是 flow-matching 模型）
                
            - 训练时随机 dropout 30% 的 advantage conditioning（用于 CFG）
                
                - Drop的作用：这样可以在推理时使用 **CFG（Classifier-Free Guidance）**进行策略改进， CFG 允许在推理时通过调整 β > 1 来锐化策略分布
                    
- 特定任务微调：
    
    - （这一步和我们现在的工作流程是一样的，采集特定的数据然后SFT微调模型）准备一些特定任务的数据，标准行为克隆SFT那套微调 pi05
        
        - 在任务特定演示数据上微调策略
            
    - DAgger 方法（迭代改进循环）
        
        - 自主 rollouts：策略自主执行收集的数据
            
        - 专家干预（Corrections）：专家在策略失败时接管并纠正的数据
            
        - 奖励反馈：每个 trial 的稀疏奖励信号
            
    - 采集好的DAgger数据，将用于微调 Value Function（critic policy），这一步和pre-train预训练是有差异的
        
        - **预训练时**：使用整个 episode 的累积奖励（N=T）
            
        - **微调时**：使用 **N=50** 的固定 lookahead, 只计算未来50步的状态值收益
            
    - 重新计算 Advantage 并更新 actor policy 策略
        
        - 这一步流程和pre-train相同
            

### GR-RL Critic Transformer（https://arxiv.org/pdf/2512.01801）

```Plain
阶段 1：离线过滤行为克隆
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ 演示轨迹      │ --> │ Q-function   │ --> │ 过滤轨迹      │

└──────────────┘     └──────────────┘     └──────────────┘
                            │                      │

┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ 轨迹          │ --> │ 镜像动作+观察 │ --> │ 增强轨迹      │
│              │     │ +翻转文本    │     │              │
└──────────────┘     └──────────────┘     └──────────────┘
                                              ↓
                                    ┌──────────────┐
                                    │ 行为克隆      │

                                    └──────────────┘

阶段 3：在线强化学习
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ 离线预训练策略 │ --> │ 在线 RL      │ --> │ 最终策略      │
│              │     │ (latent noise)│     │              │
└──────────────┘     └──────────────┘     └──────────────┘
```

- 对比上述两个，字节GR-RL就比较传统了
    
    - 模型架构
        
    - VLA 模型 π_θ + multi-task critic Q_φ， 每个Q值网络会预测一个高斯分布情况，并非预测单个Q值
        
- Q价值函数评价
    
    - 使用离线 RL + 稀疏奖励训练 Q-function
        
    - Q 值作为 Progress Function 进度函数 | Q 值自然反映任务进度
        
    - **直接用于过滤演示轨迹：** 只保留 Q 值高的 transition（对进度有正面贡献），但是你可以看到kai0和pi06都会对负样本进行条件生成，没有直接丢弃
        
        - 根据Q值 只保留对进度有正面贡献的 transition
            
        - 丢弃噪声和次优的演示数据
            
- 训练策略
    
    - 直接 BC，无 conditioning
        
    - 形态对称增强
        
        - 镜像动作和观察
            
        - 大幅提升泛化能力
            
    - 在线 RL
        
        - 学习 latent space noise predictor
            
        - 引导去噪过程朝向高回报区域

## 根据上述的思路，可以快速验证 VLA post-train的方案可以排期开发的内容需要如下：

### AWBC（优势加权行为克隆）路线 - 参考pi06和kai0

- 【该部分最早在HIL-Serl上实现类似的功能】一个GRPC服务端，用于在actor和critic之间传递transition（s，a，s‘’，r），包含了actor policy自己推理的部分，人工干预的intervention部分以及warm up用的前期自己预热的专家数据
    
- 一个稳定的模型部署 & 干预 DAgger采集失败恢复数据的流程（可以是遥操或者是原子策略）
    
- 一个更为鲁棒的VLA状态值估计器（critic policy），这个状态值估计器的预训练可以看到kai0和pi06都使用了VLA+value head的形式来进行学习
    
- 一个支持带条件生成（condition limit）的 actor policy，（对于DAgger数据的融合处理问题）
